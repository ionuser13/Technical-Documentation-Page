Introduction

HTML stands for HyperText Markup Language. It is a markup language that defines the structure of the content. It consists of a series of elements used to enclose different parts of the content so that they look or behave in a certain way. Enclosure tags play an important role here. Along with this technology, other technologies are used, mainly CSS and JavaScript.

"Hypertext" refers to links that connect web pages to one another, either within a single web site or between web sites. These are a fundamental aspect of the Web.

HTML uses markup to tag content to be displayed in the web browser. These marks contain elements, some of which we will see on this page.

Elements

See this paragraph:

              < p class="feelings">I love cats < / p>

The parts of this element consist of:

  1. The opening tag: consists of the name of the element (in this case, p), enclosed by opening and closing angle brackets (< >). Sets where the element begins or begins to take effect (in this case, where the beginning of the paragraph is)
  2. The closing tag: This is the same as the opening tag, except that it includes a slash (/) before the tag name. Sets where the element ends (in this case where the paragraph ends).
  3. Attributes contain additional information about the element that you don't want to appear in the actual content of the element. Here "class" is the name of the attribute and ""feelings" is the value of the attribute. In this case, the class attribute allows you to give the element an identifying name, which can then be used to point the element to style information and so on. They are always included in the opening tag of an element, never in the closing tag.
  4. The content: This is the content of the element, which in this case is "I love cats"
  5. The element: the opening tag and the closing tag, plus the content equals the element.

There are a lot of elements, such as heading elements, article, section, footer, tables, lists, images, video, etc.

Anatomy of a HTML Document
Elements by themselves are not useful. These have to go within a certain structure for their proper functioning. Here is the most basic structure of an HTML file: 

              < !DOCTYPE html>
              < html lang="en-US">
                < head>
                  < meta charset="utf-8">
                  < title>My test page< / title>
                < / head>
                < body>
                  < h1 >This user loves cats and animals< / h1>
                  < img src="https://metacode.biz/@test/avatar.jpg" alt="My test image">
                < / body>
              < / html>

We have the following:

  • < !DOCTYPE html >: This line of code is just needed to make sure your document behaves correctly.
  • < html>< /html>: This element wraps all the content on the entire page. Sometimes is known as the root element.
  • < head>< /head>: This element acts as a container for all the stuff you want to include on the HTML page that isn't the content you are showing to your page's viewers. This includes keywords, a page description that you want to appear in search results, CSS to style our content, character set declarations, and more.
  • < meta charset="utf-8">: This element sets the character set your document should use to UTF-8. This includes most characters from the vast majority of written languages. With this, it can handle any textual content you might put on it.
  • < title>< /title>: This element sets the title of your page, which appears in the browser tab the page is loaded in.
  • < body>< /body>: This contains all the content that you want to show on your pafe, whether that's text, images, videos, games, audio tracks, etc.
Images

Let's take a look at:

            
              < img src="https://metacode.biz/@test/avatar.jpg" alt="My test image">

This element embeds an image into our page in the position it appears. It does this via the src (source) attribute, which contains the path to our image source.

We also have the "alt" attribute, which contains a short description of the image. It is useful in occasions where users can see the image either because:

  • They are visually impaired. Users with significant visual impairments often use tools called screen readers to read out the alt text to them.
  • Something has gone wrong causing the image not to display.
Links

An HTML link is a resource that allows us to move from one page to another. We can add the hyperlink to many HTML elements, especially text or images. Here is the structure:

              
                < a href="https://testlink.com">I'm a test link< /a>

The "href" attribute is used to store the link where the "a" element redirect us. There are many more attributes for the "a" element.

Lists

HTML has special elements for making lists. At least two elements are needed to make one. The most common list types are:

  • Unordered Lists: for lists where the order of the items doesn't matter. These are wrapped in a < ul> element.
  • Ordered Lists: for lists where the order of the items matters. These are wrapped in an < ol> element.

Here is the basic structure:

              
                < ul>
                  < li>A cat< /li>
                  < li>A Dog< /li>
                < /ul>

The element "li" is used to add list items to the list. This also works on ordered lists.
